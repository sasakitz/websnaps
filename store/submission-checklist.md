### ✅ 権限の審査対応

Chrome Web Store の審査では、要求する権限すべてに正当な理由が必要です。

| 権限 | 正当性の説明 |
|------|------------|
| `activeTab` | ショートカットキーでアクティブタブをキャプチャするために必要。ユーザーの操作（ショートカット / ポップアップクリック）によってのみ有効化される |
| `scripting` | 範囲選択オーバーレイ（selector.js / selector.css）をページに注入するために必要。コアとなる機能の実装に不可欠 |
| `downloads` | キャプチャ画像を PNG/JPG ファイルとして保存するために必要 |
| `clipboardWrite` | キャプチャ画像をクリップボードにコピーするために必要 |
| `storage` | 保存形式（PNG/JPG）・保存先の設定をローカルに保持するために必要 |
| `tabs` | 対象タブの情報取得・エディタタブ作成のために必要 |
| `<all_urls>` | ユーザーが任意のWebページでキャプチャ機能を使えるようにするために必要。ページコンテンツの読み取りは行わない。スクリーンショット以外の目的では使用しない |

> **審査官向けコメント記入欄**（申請フォームの「テスト手順」欄に記載推奨）:
> ```
> この拡張機能はWebページのスクリーンショット撮影ツールです。
> <all_urls> ホスト権限は、ユーザーが任意のWebページで範囲選択オーバーレイを
> 表示するためにコンテンツスクリプトを注入する目的のみで使用しています。
> ページのコンテンツを読み取ったり、データを外部に送信することはありません。
> ```

## 申請後の流れ

1. **審査期間**: 通常 1〜3 営業日（権限が多い場合は延長あり）
2. **却下された場合**: 却下理由のメールが届く → 修正して再申請
3. **よくある却下理由**:
   - `<all_urls>` 権限の説明不足 → テスト手順欄に詳細説明を追加
   - プライバシーポリシーが不十分 → ポリシーを充実させる
   - スクリーンショットが実際の動作を示していない → 実際の操作画面を用意
